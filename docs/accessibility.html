Last updated: June 16, 2026

AshVault is designed to be playable with iOS system accessibility features. The game is fully playable muted — every audio cue has a visual and combat-log counterpart.

Supported features

VoiceOver

Combat, shop, level-up, prestige, meta screens, settings, and onboarding expose labels and hints. Decorative emoji are hidden from VoiceOver.

  • iOS Settings → Accessibility → VoiceOver

Larger Text (Dynamic Type)

Text scales through Accessibility size AX5. Layouts reflow at large sizes: combat scrolls, move buttons stack vertically, shop and meta screens use single-column cards, and headings grow instead of shrinking.

  • iOS Settings → Accessibility → Display & Text Size → Larger Text

Reduce Motion

When Reduce Motion is on, shake, bob, pulse, spring entrances, and floating combat numbers fall back to still or fade-only animations.

  • iOS Settings → Accessibility → Motion → Reduce Motion

Bold Text & Increase Contrast

Semantic text styles support Bold Text. Light and dark palettes use high-contrast surfaces for panels and inputs.

Known gaps

  • Floating combat numbers are visual-only; the combat log carries the same information.
  • Progressive unlock states for locked moves are not yet in the shipped build.
