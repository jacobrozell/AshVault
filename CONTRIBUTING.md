# Contributing & code style

Match the surrounding code first. This guide is the tie-breaker when something is ambiguous.

## Setup

See [`ios/README.md`](ios/README.md). The Xcode project is generated by XcodeGen — run `xcodegen generate` inside `ios/AshVault/` after pulling `project.yml` changes.

For Firebase telemetry in local Release builds, copy the example plist:

```bash
cp ios/AshVault/AshVault/Resources/GoogleService-Info.plist.example \
   ios/AshVault/AshVault/Resources/GoogleService-Info.plist
# Replace placeholders with your Firebase project values (never commit the real file).
```

## Architecture

- **`Models/`** — pure game logic (`GameEngine`, combat, economy, persistence). No SwiftUI imports.
- **`Views/`** — SwiftUI screens; read state from `GameEngine` via `@EnvironmentObject`.
- **`Support/`** — app infrastructure (Firebase bootstrap, analytics).
- **Docs** — living specs under `ios/docs/`; update specs and `Balance.swift` together when tuning gameplay.

## Swift style

- 4-space indentation, no file headers.
- `private` by default; widen access only when needed.
- Avoid force unwraps on dynamic values (`try!`, `as!`, `!`).
- Comments explain *why*, not *what*.

## Linting

SwiftLint runs in CI (`.swiftlint.yml` at repo root):

```bash
brew install swiftlint
swiftlint
```

## Testing

| Suite | Purpose |
|-------|---------|
| `GameEngineTests` | Combat flow, shop, save/restore |
| `PrestigeTests` | Shards, soul tree, automation |
| `ProgressionKnobTests` | Pacing constant regression lock |
| `MetaProgressionTests` | Mercenaries, relics, offline |
| `GameAnalyticsTests` | Analytics allowlist contract |
| `PlaytestTests` | Long simulation harness (run locally, not in CI gate) |

**CI scheme:** `AshVaultCI` (unit tests + code coverage on the AshVault app target).

```bash
cd ios/AshVault
xcodegen generate
xcodebuild test -scheme AshVaultCI \
  -destination 'platform=iOS Simulator,name=iPhone 17'
```

Slow playtest suites can be run alone:

```bash
xcodebuild test -only-testing:AshVaultTests/PlaytestTests
```

## Changing game balance

1. Read [`ios/docs/pacing-spec.md`](ios/docs/pacing-spec.md)
2. Edit `ios/AshVault/AshVault/Models/Balance.swift`
3. Run `ProgressionKnobTests` and optionally `python3 ios/docs/tools/balance_sim.py`

## Accessibility

See [`docs/accessibility.md`](docs/accessibility.md) for the rollout plan. New UI should use semantic text styles, `ScrollFit` for overflow-prone screens, and explicit VoiceOver labels on interactive controls.

## Pull requests

- Keep diffs focused — gameplay changes and meta/infra changes are easier to review separately.
- CI must pass: SwiftLint, build-for-testing, `AshVaultCI` unit tests, documentation summary.
